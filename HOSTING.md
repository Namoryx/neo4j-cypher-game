# 정적 배포 메모

이 프로젝트를 GitHub Pages 등 정적 호스팅에 올릴 때 발생했던 문제와 해결 방법을 정리합니다. 잊지 않고 동일한 방식으로 배포해주세요.

## 증상
- `https://<도메인>/src/main.jsx` 요청이 404로 실패함.
- 브라우저가 `text/jsx` MIME 타입으로 응답을 받아 모듈 스크립트로 로드하지 못하고 `Failed to load module script` 오류 발생.

## 해결 원리
- 루트 `index.html`에 리다이렉트를 추가해 외부(비 로컬호스트) 접근 시 자동으로 `docs/` 아래의 빌드 결과물로 이동하도록 함.
- `docs/` 폴더에는 Vite 빌드 결과(`npm run build`)가 위치하여 브라우저가 정적 자산을 올바른 MIME 타입으로 받아 사용함.

## 배포 체크리스트
1. 개발용 실행(`npm run dev`)은 루트 `index.html`이 직접 `./src/main.jsx`를 불러오므로 변경 없음.
2. 배포 전 `npm run build`를 실행하면 최신 번들이 `docs/`에 생성됩니다.
3. 정적 호스팅 루트는 저장소 루트이며, 외부 사용자는 자동으로 `docs/`로 리다이렉트됩니다. `docs/` 안의 `index.html`도 동일한 리다이렉트 로직을 포함합니다.
4. 배포 후에도 같은 오류가 보이면 `docs/`가 최신 빌드인지, 리다이렉트 스크립트가 삭제되지 않았는지 확인하세요.
